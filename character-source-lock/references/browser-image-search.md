# Browser image search / Google Lens workflow

Use this when a user asks to identify a character from an image, especially anime/game/fandom avatars.

## Preferred path

1. Open Google Images / Google Lens in a browser.
2. Upload or drag the source image into image search.
3. Read the result categories:
   - AI Overview / About this image
   - Exact matches
   - Visual matches
   - source pages under the image
4. Extract candidate name/franchise from repeated high-signal results.
5. Click/open at least one profile/wiki/source page to verify.
6. Click/open at least one image/gallery/result page if visual details are needed.
7. Save source-lock note with source URLs and confidence.

## Example: Hikari avatar

A Google image search / Lens result on Hikari's avatar surfaced:

- `This image features Mythra, a main character from the action role-playing game Xenoblade Chronicles 2.`
- It also surfaced related visual matches such as Mythra/Pyra boards and pixiv-style results.

This was stronger than generic text search because the image itself carried the character identity.

## What to trust

Trust order:

1. Exact/visual match repeated across multiple results.
2. Official/wiki/database page confirming character identity.
3. Gallery/sprite/source pages confirming visual anchors.
4. Sticker/GIF pages only for pose/expression conventions.

Do not trust a single AI Overview blindly. Use it as a lead, then verify with source pages.

## When to stop

Stop after the minimal 3-source set if confidence is high:

- profile/source page
- visual/gallery/sprite page
- sticker/pose convention page if needed

Expand only if candidates conflict.
