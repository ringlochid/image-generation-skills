# Telegram sticker examples

For client/commercial work, prefer the bundled generic non-IP style asset unless the user explicitly accepts rights/source risk for public screenshots.

Use these as style references, not assets to copy. Extract visual grammar only: scale, crop, outline, expression vocabulary, prop use, and pack consistency.

## How to use these refs

Use them in two levels:

1. **Style grammar**: read/inspect pages and extract crop, outline, scale, expression vocabulary, prop grammar, and pack consistency.
2. **Visual references**: when exact style matters, capture/save a small curated screenshot/contact sheet from accessible public pages and pass that image as a style-only reference beside the identity image. Do not copy exact poses or art.

## Official technical baseline

- Telegram sticker import docs: https://core.telegram.org/import-stickers
  - Verified accessible 2026-04-26.
  - Static stickers: PNG/WebP with transparency, max 512 KB, max 512x512 px, one side must be 512 px, white stroke/shadow recommended.
- Telegram sticker API docs: https://core.telegram.org/api/stickers
  - Verified accessible 2026-04-26.
  - Static stickers are WebP; one side must equal 512 px; transparency is supported.

## What to extract

- Pack-level consistency: same character, same scale, same outline weight.
- Chat readability: big heads/faces, low detail, strong silhouettes.
- Emotion vocabulary: exaggerated face first, props second, decorative marks third.
- Prop grammar: laptop/food/magnifier/pillow work if they are simple and text-free.
- Export target: individual 512px stickers with transparent background and white stroke/shadow.

## What not to copy

- Exact official/fan poses.
- Character-specific art, outfits, props, or stickers from copyrighted/fan packs.
- Watermarks, channel labels, signatures, pack logos, or text overlays.
